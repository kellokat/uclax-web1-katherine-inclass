01. Explain the difference between HTML, CSS and JS.

  • HTML, CSS, and JS are used together to create websites. HTML stands for Hyper Text Markup Language. It is a programming language used for webpages. Web browsers take the markup and use it to display text, images and content in a web page. It is used to specify the content of web pages.
  • This is different from CSS. CSS is the Cascading Style Sheet. An HTML document is styled using this language. In other words, the CSS contains the styling language for the web page. Each attribute has its own title and can be specifically written to produce specific colors and fonts. It is also used to configure layouts.
  • JS deals with scripting. HTML and CSS don’t. JS stands for Javascript. It is a scripting programming language for the web and deals with the behavior of the website. Content created with it runs on the web browser. Without Javascript, most of the content or functionality on the web page would be useless. It is the scripting programming language that creates dynamic content for a web site.
  • While HTML specifies the content, CSS specifies the styling. JS is used to program the behaviors and actions of the web page. They are used together to create a website.

02. What are single page apps (SPAs)? How do they differ from traditional multi-page Web Sites?

  • Single page applications, also known as SPAs, run on one page of the website. Code is rewritten based on the user’s choices within the webpage. The advantage of single page applications is that the content does not have to reload every time the user makes a selection.
  • All sources, from HTML to JS and CSS and any other data from the web server is loaded instantly, while the page itself does not reload.
  • Websites, therefore, feel more like native applications, rather than the traditional loading of basic HTML, JS and CSS. This gives the website a dynamic feel and enhances the user’s experience.
  • In multi-page websites, the user clicks links that navigate to different pages and subpages. This means new data is loaded in a brand new page. All the content must reload each time the user clicks a link.

    • What it means for a company to be Agile. What is the purpose of using Agile, and its Pros and Cons.

  • Agile is used in software development. For a company to be Agile, there must be a team who works together to get the project done. Rather than working on the whole project at one time, sections, also called Sprints, are dealt with so that the project moves forward efficiently. The work is iterative, lightening the amount of re-work that takes place.
  • Agile development creates better control over the project. It also reinforces team efficiency, flexibility, adaptability,and scalability. Cost estimates are more precise and the project and the team are always evolving. Change can easily take place at any time.
  • One disadvantage is that everyone needs to be on the same page. If not, bad behaviors can make the project suffer. Another is less predicatability, if you are working on small facets of the project at one time, sometimes the overall view becomes foggy. Agile Development also takes more time and commitment with a lot of demand on the team and the client. And the project can also lose it's focus and fall off track.

    • Define the following roles: Project Manager, Business Analyst, Scrum Master, UX designer, Web Developer/Engineer, Quality Assurance Tester, and DevOps.

  • The Project Manager oversees every aspect of the work that is being done on the project. The Project Manager plans, procures and executes the project's goals.
  • The Business Analyst is a person in a business or organization that analyses, records and documents the ongoing business processes. They use data analysis to improve the project.
  • The Scrum Master uses the Scrum Guide to establish Scrum, which is an agile framework for software development for teams to work together. They use their knowledge of Scrum to guide the Scrum Team and organization. He or she manages the team and is held accountable for their productivity.
  • The UX designer determines the experience of the user, and makes the product useful. They have many responsibilities such as product research, creating personas, idenitfying key user groups, Information Architecture, creating wireframes, prototyping, and product testing.
  • The Web Developer/Engineer is a programmer for the creation of websites and apps on the internet. They create applications that use HTML, CSS and JavaScript. And in the server, they use PHP, ASP.NET, Python, Go or Java. For the client and the server to communicate, they use http.
  • The Quality Assurance Tester ensures the level of quality of the app or website that is being developed. They focus on making the software development process better and making everything work under the quality standards of software products. They test the app and detail any bugs they find.
  • DevOps stands for development and operations. People, processes, and technology are united together to provide a valuable experience for the user. Teams using DevOps create better apps and websites for the user's experience. It's the engineer who handles the architecture and deployment.

    • What is the difference between jpg, gif, png and SVG images?

  • A jpg is a digital image that uses lossy compression to reduce file size. While most digital images achieve 10:1 compression, the image quality is not greatly reduced. Jpg is the most popular image commpression and is used throughout the internet.
  • A gif is a bitmap image format, supporting 8 bits per pixel and has its own color palette using up to 256 colors. Gifs use Lempel-Ziv-Welch lossless data compression. The images are not realistic and are used for graphics or logos.
  • PNGs have lossless data compression and are a raster-graphics file format. It was developed to replace GIFs. PNGs can have transparent backgrounds and can use palettes of 24-bit RGB or 32-bit RGBA colors.
  • SVG images are used for 2d graphic and use an Extensive Markup Language (XML)-based vector image format. XML text files determine the SVG image's behaviors. A standard text editor can be used to create and edit the SVG images. There are three types of graphic objects: vector graphic shapes, bitmap images and text and they use CSS and JavaScript. Data compression varies, but SVG images can achieve lossless data compression.
